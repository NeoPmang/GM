✨ 프로젝트 목표 & 핵심 일정

웹샵의 최우선 목표는 모바일 및 PC 게임 사용자 모두에게 플랫폼 제약 없이 효율적인 아이템 구매 채널을 제공하는 것입니다.

  • 초기에는 모바일 게임 사용자를 대상으로 집중합니다.
  • 향후 PC 게임으로 확장할 계획입니다.

웹샵 오픈 목표일은 9월 23일경으로, 월말 결제 이슈나 월초 매출 집중 문제를 피하기 위해 웹보드 점검 일정과 동기화합니다.

  • 2026년: PC 피망 웹샵 노출 확장

플랫폼개발1팀
2025년 4월 29일: M.pmang 로그인 세션 유지 및 로그인 정보 가져오는 API 준비 완료. 웹샵만을 위한 로그인 API 구현은 공수가 크지 않을 것으로 확인.
웹샵 구조
  • 각 게임별(고스톱, 포커, 뉴맞고, 쇼다운홀덤, 섯다) 별도 운영.
  • 포커몰 구조 최대한 활용하여 개발 효율성 증대.
  • 도메인: shop.pmang.com 하위 게임별 구분 (예: shop.pmang.com/gostop) 우선 고려.
결제 환경
  • 모바일 결제 환경 고려, PG사 모듈 호출 시 PC/모바일 환경 구분 필수.
  • 유저 에이전트 또는 화면 크기 사용 환경 구분.
  • 정확한 구분 어려운 경우 모바일 버전을 PC 환경에 띄우는 방안도 고려.
API 연동
  • 웹샵은 프론트엔드 + Backend for Frontend 역할, 각 게임별 게임홈과 통신하여 데이터 획득.
  • 게임홈: 게임 API와 통신하는 API Gateway 역할.
  • 상품 정보: NIMS → NSIS 연동.
  • 유저 정보: GPM 연동.
  • 결제(빌링 UI) URL: 빌링 시스템 제공.
  • 결제 완료 후 효력 지급: 웹샵으로 결과 리다이렉트.

📊 상품 관리 & 회계

상품 정보
상품 정보(이미지 및 카테고리 제외)는 NIMS에서 가져옴. 상품 이미지 및 카테고리 정보는 웹샵 어드민에서 별도 등록 및 관리.
레드/블루 기획팀
  • 2025년 5월 21일: 보너스 라벨 및 웹샵 홈 IP 구분 노출 이미지 처리 확정. 상품 관리는 NIMS, 웹샵 어드민은 상점 관리 기능만 사용.
  • 2025년 6월 16일 (레드 기획팀 - 박기현): "뉴맞고" 웹샵 판매 예정 상품 리스트 공유. 모바일 상점과 동일 상품을 더 높은 요율로 판매 계획.
  • 2025년 6월 18일 (블루사업팀 - 최기석): "피망 포커", "쇼다운홀덤" 판매 예정 상품 리스트 공유. 기존 루비/다이아, 웹샵 전용 신규 아바타 포함. 아바타 상품 정보 변동 가능성 있음.
재논의 결과: PC 피망 로그인/결제 사용으로 결정.
  • 기존 M.pmang/PC 피망 미사용 모바일 사용자 접근성 문제 제기.
  • PC 피망 로그인 모바일 세션 유지 문제 재확인.
  • PC 충전창 모바일 호출 시 UI 불편/노후화 이슈는 빌링 개편 전까지 허용.
  • PC 빌링 사용 시 다양한 PC 결제 수단 제공 및 백엔드 작업 공수 감소 이점.
  • 세션 끊김 리스크는 우선 감안 후 오픈, 세션 유지 이슈 추후 재검토.
내부회계팀
2025년 4월 22일: 웹샵 자체는 내부통제 범위 미포함, 상품 관리 기존 프로세스 유지 확인. 단, 재구매일수(회계매출) 산출 방식 변경 시 내부통제 수정 필요, 회계1팀과 논의 필요.
회계1팀
  • 2025년 4월 21일: NSIS 충전 및 매출 집계, 기존 피망과 결제/구매 데이터 및 상품 구조 동일 시 문제 없음. 데이터 구분 필요 시 SITE ID, SVC CODE 다르게 설정 가능성. 웹샵 구매는 숨겨진 캐시 충전/구매 → 피망 캐시 사용 구조, 모바일/PC 사용자 모두 별도 상점 적용.
  • 2025년 4월 22일: NSIS에서 NIMS 스토어 구분 정보별 NBOS 상품 판매 매출 정보 구분 기존 로직 불가, 신규 매출 인식 프로세스 구성 필요.
  • 2025년 4월 23일:
    • NSIS 매출 정보 내 웹샵 매출 구분 가능하나, 표시 카테고리 기존 PC 상품과 동일 (NBOS 상품당 매출 카테고리 1개 제한). 변경 시 구조 변경 필요.
    • 웹샵 매출은 모바일/PC 유저 구분 어려움 (PC 유저도 접근/구매 가능).
    • NIMS에 웹샵 전송 기능, NBOS에 웹샵 노출 카테고리 추가 필요.
    • SVC_Code와 Site SRL은 기존과 동일 유지하여 부작용 방지.
    • 김형근 제안: 노출 카테고리 대신 매출 카테고리를 게임별로 새로 만들어 구분 → 웹샵용 상품 추가 등록.
    • 박정수 동의: NSIS 매출 카테고리 문제 해소, 웹샵 내 게임별 상품 구분 및 고객 지원에서 NBOS 거래 내역 확인 위해 노출 카테고리도 추가 방향 진행.
경영인프라팀
2025년 4월 21일까지 NSIS 관련 구조 정리 완료 예상.
